一、使用的限制 ​

raina-schematic 接受的数据和生成的数据都有以下默认的限制范围。

  1. NodeProto 决定了 Node 展示的尺寸，不依赖其他数据
  2. Port 分为 inputPort 与 outputPort，挂载在 NodeProto 上，支持使用 Port 和 NodeProto 提供的 api 自定义特定 Port 的坐标位置，当不进行自定义时，默认 inputPort 会挂载在 NodeProto 左侧，outputPort 会挂载在 NodeProto 右侧。Port 依赖 NodeProto。
  3. Node 是 NodeProto 的实例，一个 NodeProto 允许对应多个 Node。Node 依赖 NodeProto。
  4. Edge 是两个 Node -> Port 之间的连线(因为 Port 只与 NodeProto 关联，所以要得到一个视图层面的 Port，需要 Node -> Port 的数据才足以关联)，其中一个 Port 的类型必须是 inputPort，另一个 Port 的类型必须是 outputPort
